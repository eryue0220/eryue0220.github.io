# weekly

人真的很奇怪，以前在公司不喜欢写周记，觉得特别烦。现在反而习惯性地开始让自己写起了周记，希望能够保持。

虽然是不上班了，但是不能工作。这周已经开始在准备简历和面试了，说是这周，但其实已经是两个月前了。毕竟已经两个月没有上班了。这次找工作坚定地要去尝试下外企或者是 remote 的单位，体验了没有通勤的日子，实在是太快乐。这周投简历还是冲动了一些，虽然拿到了一家的面试。但是投了两三家自以为能获得面试的公司，却都没有消息后就开始投一些压箱底的公司，万一突然进入面试还真没有多大把握能够拿下。毕竟才刚刚开始准备，算法题也才刚刚开始刷起来。而算法真的是一个让人感到痛苦的学习内容，每每到要准备面试，面对算法题内心总有十万分的拒绝。这周进度略让自己感到满意，总算是沉下心来开始阅读算法，也算是开始进入了学习节奏。但是还是很容易忘记，感觉一天不看，第二天自己就不太记得了。周末两天会继续把树的部分给看完，同时开始针对性地做下后面的练习题，然后配合 neetcode 上把 tree 相关题目给刷完。不过这周应该是做不完，争取到下周末能把这个目标实现。

因为是面的外企，所以口语、听力也是一个锻炼的关键。口语听力虽然一直都听说能用 chatgpt 来辅助练习，但于我而言的效率有些低，因为根本不知道要跟他说些什么才能聊起来。听力倒是最近可以听 30% ~ 40% 的 ted 演讲了，而且专注听的话，有时候甚至能有 50% 的内容是能够听懂和理解，这算是这两个月来自己为数不多的长进吧，下周开始，争取每天早起都在扇贝听力上听一到两篇 ted 演讲（又一个 flag 高高举起）。至于写呢，因为平时工作关系，大部分的内容都是用英语来写，基本来说符合要求吧。为什么是基本呢？因为最近在看 twitter 的时候就感觉，上面的很多内容和表达都非常地口语化，自己写的反而太过正式，不像是一个 native speaker，而很多口语化、或者称为本地化的表达自己却反而难以理解。这周也给自己在英语上定了一个目标（从前司出来，多少都被感染了😂🥶），参加十二月中旬的雅思考试（UKVI），目标是 6.5 分，争取能有7分。这周背单词，还发现了个好工具 [QWERTY Learner](https://qwerty.kaiyi.cool/)，可以帮助自己记雅思单词和短语。

这周之所以能沉下心来开始认真准备算法，因为之前把大部分精力都放在了开源项目上。妄想着要靠开源项目的贡献能够对自己的面试有所帮助，但后来仔细想想，不过是自己的妄想罢了，毕竟才刚开始的这两个月，也才刚开始参与一些项目，甚至都不是项目的 `collaborator`，应该是机会渺茫。所以还是决定沉住气来好好准备面试。

因为心态上的调整，也可能是慢慢地参与到几个项目中后，自己也不在像之前那么疯狂。几乎每周都逼自己看两三个开源项目，每个项目看两三天，如果自己不太熟悉、不喜欢或者是觉得自己的能力没啥能够参与的就放弃了，而精力和专注力反而在这样项目切换上被逐渐消磨殆尽。接下来的项目，自己还是希望能够更聚焦一些，在坚持给 `jest` 贡献外，主要围绕在编译器相关方向，顺便看看能否深度参与到 `babel`、`biome` 以及 `esbuild` 相关项目上。体验了跨多语言工作，感觉比较愉快，至少一个语言项目上看厌烦了，可以切换到另一个看看，也算是换种思路。

本周还开始了一个新技能，开始学习游泳了。从小算是在海边长大的人，第一次开始学习游泳。体验就是还不如幼儿园小朋友游得好，甚至遭受到了来自小朋友的质疑。但是不管多大，玩水它总归是快乐的，去游泳的这一个半小时里，泡在水里让我也暂时放下思考，毕竟稍一慌神，就要喝水了。

感觉好像别人开学了我也开学了。

开源项目上，本周在 Jest 上水了两个 PR，[14459](https://github.com/jestjs/jest/pull/14459) 和 [14460](https://github.com/jestjs/jest/pull/14460)，两个 PR 因为是 breaking change。都将会在 `Jest` 的 30 版本才会被 merge。按目前的 milestone 来看，v30 版本还需要一段时间。 本周也给 `eslint-plugin-jest` 提供了一个新规则 `no-confusing-set-timeout`，[pr#1425](https://github.com/jest-community/eslint-plugin-jest/pull/1425)，算是初步完成上周自己定的其中一个目标，深入学习 `eslint`，读了下 eslint 核心的 codePath 相关代码，也不是很理解，为啥 [code path analysis](https://eslint.org/docs/latest/extend/code-path-analysis) 这么核心的文章没有放出来在官网上，要不是在网上搜了下，估计自己还得走不少弯路。争取这个月能给 `eslint`提交一到两个 bug fix 相关的 PR。
