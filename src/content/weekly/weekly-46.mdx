# Weekly 46

拖更了两周了，前面两周一边面试一边又在准备英语以及写代码等等（不过还是因为懒）。这周想了想还是写吧。保持写周记也算是给自己一个记录，方便之后给自己一个回顾和总结，让自己感觉这段时间没有白过（一些些无用的自我安慰）。

先说开源吧。这两周的开源项目，花了不少时间在阅读代码上。主要有以下几个项目：

- [nest](https://github.com/nestjs/nest)
- [grafana](https://github.com/grafana/grafana/tree/main)
- [opentelemetry-js-contrib](https://github.com/open-telemetry/opentelemetry-js-contrib)
- [opentelemetry-js](https://github.com/open-telemetry/opentelemetry-js)
- [nerdctl](https://github.com/containerd/nerdctl)

基本上大多数项目都围绕着 Golang 和云相关的内容，这也是明年自己计划打算要深入的领域吧，虽然跨度有点大，但也算是给自己设立的一个明年的新 flag 吧。争取能够在云相关的项目上能有一些 contributions。

至于在开发上，这两周还算是比较有收获的吧。完成了第一个 apache 的 [PR](https://github.com/apache/incubator-opendal/pull/3297)，虽然过程几经波折，但最终还是成功被合并了。 这次过程当中自己也学会很重要的一点是，遇到有模糊的地方自己还是需要先提出来，把问题或者方案先理清楚。因为可能这个过程中协作方可能一开始也不会考虑得那么全面。提交代码也算是一种吧，就是需要来回改，从这个 PR 的 commit 和 comments 次数也可见一斑。其次就是这个 PR 也让学到了很多 CI 的开发技巧，有不少 yaml 文件的更改都直接照着其他的配置 cv 一份来进行修改。也给 jotai 提交了一个 bug fixed [PR](https://github.com/pmndrs/jotai/pull/2197)，整个过程也是挺有趣的，一开始我还不太了解 jotai 的代码库，尝试地给作者提了个 PR，作者一开始也留言表示，可能不一定相关，同时这个代码还会导致其他测试 case 出现问题，可过了几天我再看时，就已经被 merge 了。以及还给 oxc 提交了三个 jest 相关的 linter 的 feature。

前面也提到了明年有计划要系统地学习云相关的知识，所以这半个月也抽时间把 [Mastering Go](https://book.douban.com/subject/33474342/) 读完了，入门的话还算不错吧，感觉比 《The Go Programming Language》讲的更简单一点，起码上手没难度，跟着敲一遍代码就大致知道大概什么意思了。作者的讲解也还算是细致。但也仅限于入门，如果已经入门了 Go 语言，那这本书就太简单了。接下来的这两个月里吧，除了保持开源项目的投入外，还想着让自己花些时间投入到 Golang 和 C/C++ 的学习上，趁着之前 JD 打折，入手了：

- 《C++之旅》
- 《Go语言设计与实现》
- 《GO专家编程》
- 《Go语言精进之路》

而至于为啥要学 C/C++ 呢，因为我也还想着能去读一读 Node.JS 相关代码，毕竟自己每天都在用，得要花时间去看看。

英语准备上，这两周的进度算一般吧，单词倒是放了一段时间了。这段时间主要是在练雅思听力和雅思阅读上，听力练下来感觉以前考四六级的手感还在，没有丢掉太多，就是波动有些大。做了几篇真题，大概在 25 ~ 30 分的样子，有时候自己一着急，就 22 ~ 24。感觉时间还是够的，只是保持一定的阅读速度即可。听力的话，就真一般了。很容易听完后面忘了前面，导致写题的时候，一般靠听一般靠猜。下周的话要把英语单词重新捡回来了，同时要认真地练习下写作和口语了，在 12 月份考试前再突击下，保 6 争 7。
